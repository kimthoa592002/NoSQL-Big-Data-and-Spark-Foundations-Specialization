## Hadoop Multi-Node Cluster Setup (Optional) - Coursera Cloud IDE
Environment:
- Platform: Coursera / Cognitive Class / Skills Network
- Tool: Cloud IDE (Theia-based), Docker, Docker-compose
- Framework: Apache Hadoop 3.2.1 (Multi-container cluster)
= Access: Bash into NameNode, Web GUI (Port 9870)

---

### 1. Setup Hadoop Cluster using Docker

***+ Open a new terminal***

Navigate to Terminal -> New Terminal.

***+ Clone the repository: Copy the Docker configuration to your environment***

git clone [https://github.com/ibm-developer-skills-network/ooxwv-docker_hadoop.git](https://github.com/ibm-developer-skills-network/ooxwv-docker_hadoop.git)

***+ Navigate and Build: Move to the directory and start the multi-container application***

cd ooxwv-docker_hadoop
docker-compose up -d

***+ Verify Containers***

Docker-compose will start 5 containers: namenode, datanode, resourcemanager, nodemanager, and historyserver.

***+ Access NameNode: Enter the NameNode container using bash***

docker exec -it namenode /bin/bash

### 2. Explore Configuration Files

#### The Hadoop environment is configured via specific XML and shell files located in /opt/hadoop-3.2.1/etc/hadoop/:

+ hadoop-env.sh: Main file for YARN, HDFS, and MapReduce settings.

+ core-site.xml: Defines core properties for HDFS and Hadoop.

+ hdfs-site.xml: Manages metadata storage, HDFS images, and logs.

+ mapred-site.xml: Contains MapReduce configuration parameters.

+ yarn-site.xml: Defines YARN settings for Resource and Node Managers.

### 3. HDFS File Operations

***+ Create Directory: Setup the standard user input path in HDFS***
hdfs dfs -mkdir -p /user/root/input

***+ Copy Config Files: Upload local XML configs to HDFS***
hdfs dfs -put $HADOOP_HOME/etc/hadoop/*.xml /user/root/input

***+ Download Sample Data: Fetch a text file for testing***
curl [https://raw.githubusercontent.com/ibm-developer-skills-network/ooxwv-docker_hadoop/master/SampleMapReduce.txt](https://raw.githubusercontent.com/ibm-developer-skills-network/ooxwv-docker_hadoop/master/SampleMapReduce.txt) --output data.txt

***+ Upload to HDFS: Move data.txt to the HDFS root directory***
hdfs dfs -put data.txt /user/root/

***+ Verify Upload: Read the content of the file directly from HDFS***
hdfs dfs -cat /user/root/data.txt
